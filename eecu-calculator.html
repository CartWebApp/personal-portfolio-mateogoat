Overview

This application allows a user to calculate their monthly expenses and find out if they are over budget or under budget, depending on the career they select. Every career has a different net income, which is the fundamental reason this budget calculator was created.

Concepts Learned

  • Cross-functional Team Leadership: A core portion of this project was to have my colleagues code the design that I provided to them, and coaching them from some being very skilled and others not being is a core skill needed while working for a company.
  • Chart.js: To display a doughnut chart to the user in real time as they are inputting their monthly expenses, Chart.js is required to display this. It shows the different categories and updates every time a user inputs any new input.
  • Fetch API: In order to show the list of careers a user may have, the fetch API is required to bring those careers from a dataset to the front-end of the webpage.

Problem Solutions

  • The Problem: I struggled with how to use Chart.js with the code my peers had done previously, which was a fundamental portion required for the client.
  • The Solution: I received assistance from a very intelligent individual who spent over an hour finding a solution to the problem

Habits of Mind

  • Persistence: Even when others were struggling to implement my design into a functional website, I kept pushing them to the best of their ability by coaching them on what to fix with their code.
  • Receiving Feedback: After I submitted my work, I received feedback from my instructor and my peers on what I should update on my website. I documented this feedback and am currently working on alterations to the code to allow for a smoother user experience.
